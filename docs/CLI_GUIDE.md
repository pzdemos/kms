# KMS CLI 使用指南

KMS CLI 是一个交互式命令行工具，用于管理密钥、项目、用户和审计日志。

## 安装

### 通过 npm 安装

```bash
npm install -g @pengzi/kms
```

### 从源码安装

```bash
# 克隆项目
git clone <repository-url>
cd kms

# 安装依赖
npm install

# 构建项目
npm run build
npm run build:cli

# 全局链接
npm link
```

## 配置

KMS CLI 需要连接到 MongoDB 数据库。你可以通过以下三种方式配置：

### 方式 1: 配置文件（推荐）

创建配置文件 `~/.kms/config.json`：

```json
{
  "connectionString": "mongodb://localhost:27017",
  "databaseName": "kms"
}
```

### 方式 2: 命令行参数

```bash
kms --connection "mongodb://localhost:27017" --database "kms"
```

### 方式 3: 交互式配置

启动 CLI 后，选择 `系统设置` → `切换连接` 来配置数据库连接。

## 功能详解

### 1. 项目管理

#### 创建项目

1. 选择 `项目管理` → `创建新项目`
2. 输入项目名称
3. 设置主密码（用于加密项目密钥，至少 8 个字符）
4. 确认创建

**重要**: 主密码用于加密项目的所有密钥，请妥善保管！

#### 查看所有项目

选择 `项目管理` → `查看所有项目`，会显示所有项目的列表：
- 项目 ID
- 项目名称
- 状态
- 元数据

#### 删除项目

选择 `项目管理` → `删除项目`，然后选择要删除的项目。

**警告**: 删除项目会删除项目下的所有密钥，此操作不可恢复！

### 2. 密钥管理

#### 创建密钥

1. 选择 `密钥管理` → `创建密钥`
2. 选择项目
3. 输入密钥信息：
   - 密钥名称
   - 密钥类型（mongodb、mysql、postgresql、redis、custom）
   - 密钥值（连接字符串）
   - 描述（可选）
   - 标签（可选，逗号分隔）
4. 输入项目主密码
5. 确认创建

#### 查看所有密钥

1. 选择 `密钥管理` → `查看所有密钥`
2. 选择项目
3. 查看密钥列表（不包括密钥值）

#### 获取密钥值（解密）

1. 选择 `密钥管理` → `获取密钥值`
2. 选择项目
3. 选择密钥
4. 输入项目主密码
5. 查看解密后的密钥值

**安全提示**: 密钥值会在终端显示，请确保周围没有他人！

#### 更新密钥

1. 选择 `密钥管理` → `更新密钥`
2. 选择项目
3. 选择要更新的密钥
4. 输入新信息（描述、标签）
5. 输入项目主密码

#### 删除密钥

1. 选择 `密钥管理` → `删除密钥`
2. 选择项目
3. 选择要删除的密钥
4. 确认删除

#### 搜索密钥

1. 选择 `密钥管理` → `按标签搜索`
2. 选择项目
3. 输入搜索关键词
4. 输入标签过滤（可选）

### 3. 用户管理

#### 创建用户

1. 选择 `用户管理` → `创建用户`
2. 选择项目
3. 输入用户名和密码
4. 选择角色：
   - `admin` - 管理员（所有权限）
   - `operator` - 运维人员（读取、更新密钥）
   - `developer` - 开发人员（读取密钥）
   - `readonly` - 只读用户（列出密钥）
   - `auditor` - 审计员（查看日志）

#### 查看用户列表

选择 `用户管理` → `查看用户列表`，可以查看项目的用户统计信息。

#### 删除用户

选择 `用户管理` → `删除用户`。

**注意**: 此功能需要直接在 MongoDB 中操作，CLI 会提供相应的命令。

### 4. 审计日志

#### 查看最近日志

1. 选择 `审计日志` → `查看最近日志`
2. 选择项目
3. 查看最近的操作记录

日志包含：
- 时间戳
- 操作类型
- 操作状态（成功/失败）
- 操作者

#### 搜索日志

1. 选择 `审计日志` → `搜索日志`
2. 选择项目
3. 输入操作类型过滤（如 `key.create`、`user.login`）

### 5. 系统设置

#### 切换数据库连接

1. 选择 `系统设置` → `切换数据库连接`
2. 输入新的连接字符串
3. 输入数据库名称
4. 系统会测试连接是否成功
5. 选择是否保存配置

#### 生成加密密钥对

1. 选择 `系统设置` → `生成加密密钥对`
2. 输入私钥密码（可选，推荐设置）
3. 系统会生成 RSA-4096 密钥对

**输出内容**:
- 公钥（可存储在代码库中）
- 私钥（请妥善保管，不要提交到代码库）

#### 查看当前配置

选择 `系统设置` → `查看当前配置`，显示当前的数据库连接信息。

## 权限系统

KMS 使用基于角色的访问控制（RBAC）：

| 角色 | 权限 |
|------|------|
| admin | 所有权限 |
| operator | 创建、读取、更新密钥 |
| developer | 读取密钥 |
| readonly | 列出密钥 |
| auditor | 查看审计日志 |

## 安全最佳实践

1. **主密码**: 使用强密码，至少 8 个字符，包含大小写字母、数字和特殊字符
2. **API 密钥**: 创建用户时显示的 API 密钥只显示一次，请妥善保管
3. **RSA 密钥对**: 私钥应该存储在安全位置，不要提交到代码库
4. **审计日志**: 定期检查审计日志，监控异常访问
5. **访问控制**: 为不同用户分配合适的角色，遵循最小权限原则

## 配置文件位置

- **Linux/Mac**: `~/.kms/config.json`
- **Windows**: `%USERPROFILE%\.kms\config.json`

## 故障排除

### 连接失败

1. 检查 MongoDB 是否正在运行
2. 验证连接字符串是否正确
3. 检查网络连接和防火墙设置
4. 确认 MongoDB 用户权限

### 权限错误

1. 确认当前用户是否有权限执行操作
2. 检查用户角色是否正确
3. 联系管理员分配适当权限

### 主密码错误

主密码用于加密/解密密钥，如果忘记：
- 无法恢复加密的密钥
- 需要重新创建项目
- 建议将主密码存储在安全的密码管理器中

## 示例工作流

### 快速开始

```bash
# 1. 启动 CLI
kms

# 2. 创建项目
选择: 项目管理 → 创建新项目
项目名称: my-project
主密码: MySecurePassword123!

# 3. 创建密钥
选择: 密钥管理 → 创建密钥
项目: my-project
密钥名称: production-db
密钥类型: mongodb
密钥值: mongodb://user:pass@host:port/db
标签: production,db

# 4. 获取密钥
选择: 密钥管理 → 获取密钥值
项目: my-project
密钥: production-db
主密码: MySecurePassword123!
```

### 团队协作

```bash
# 1. 管理员创建项目
kms → 项目管理 → 创建新项目

# 2. 创建团队成员用户
kms → 用户管理 → 创建用户
角色: developer

# 3. 开发人员使用自己的账户
kms → 设置当前用户 → developer-id

# 4. 查看审计日志
kms → 审计日志 → 查看最近日志
```

## 高级功能

### 使用加密连接字符串

如果你的连接字符串需要加密存储：

1. 生成 RSA 密钥对：`系统设置` → `生成加密密钥对`
2. 使用公钥加密连接字符串
3. 将加密后的连接字符串存储在配置文件中
4. 私钥通过环境变量或安全存储提供

详细说明请参考 [加密配置指南](./ENCRYPTION.md)。
