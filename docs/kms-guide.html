快速开始

5 分钟完成 KMS 部署和第一个密钥创建

01

安装 KMS

使用 npm 全局安装或作为项目依赖

# 全局安装（推荐 CLI 使用）
npm install -g @pengzi/kms

# 项目依赖安装
npm install @pengzi/kms
02

连接数据库

配置 MongoDB 连接，生产环境建议使用 MongoDB Atlas（自动 TLS）

const { KMSClient } = require('@pengzi/kms');

const kms = new KMSClient({
  connectionString: 'mongodb+srv://user:pass@cluster.mongodb.net/kms',
  databaseName: 'kms'
});

await kms.connect();
03

创建项目

创建项目并设置主密码，主密码用于派生项目主密钥

kms.setCurrentUser('admin-user');

const project = await kms.createProject(
  'my-app',
  'StrongPassword123!',
  { environment: 'production' }
);
04

创建密钥

创建第一个加密密钥，支持多种密钥类型

const { KeyType } = require('@pengzi/kms');

const key = await kms.createKey(
  project.projectId,
  'StrongPassword123!',
  {
    keyName: 'mongodb-primary',
    keyType: KeyType.MONGODB,
    value: 'mongodb://user:pass@localhost:27017/db',
    tags: ['production', 'database']
  }
);
05

获取密钥

需要时解密获取密钥值用于应用配置

const keyValue = await kms.getKey(
  project.projectId,
  'StrongPassword123!',
  key.keyId
);

console.log('密钥值:', keyValue.value);
06

断开连接

使用完毕后断开数据库连接

await kms.disconnect();
✓ 部署完成 恭喜！你已经完成了 KMS 的基本使用流程。接下来可以探索更多功能：用户管理、审计日志、CLI 工具等。

安全机制

多层安全防护体系，确保密钥全生命周期安全

🔐

第 1 层：加密存储

AES-256-GCM 加密算法，PBKDF2 密钥派生（100,000 迭代）

  • 每个项目独立主密钥
  • 密钥值永远不以明文存储
  • 加密包含完整性验证
  • 密钥泄露只影响单个项目
👮

第 2 层：访问控制

基于角色的访问控制（RBAC），支持 5 种角色权限

  • Admin - 所有权限
  • Operator - 创建、读取、更新密钥
  • Developer - 读取密钥
  • Readonly - 列出密钥
  • Auditor - 查看审计日志
📝

第 3 层：审计日志

完整记录所有操作，支持安全监控和合规审计

  • 所有密钥操作记录
  • 用户操作追踪
  • 权限拒绝事件
  • 时间戳和操作者信息
🌐

第 4 层：传输安全

支持 TLS/SSL 加密传输，防止中间人攻击

  • MongoDB Atlas 自动 TLS
  • 自建 MongoDB TLS 配置
  • 证书验证和主机名检查
  • 防止数据窃听和篡改
🔑

第 5 层：密码安全

Bcrypt 哈希存储，API 密钥安全生成

  • Bcrypt 加密（salt rounds: 10）
  • API 密钥哈希存储
  • 主密码强度要求
  • 建议定期轮换（180 天）
