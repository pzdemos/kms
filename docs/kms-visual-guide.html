5 分钟快速开始

从安装到第一个密钥的完整流程

1

安装 KMS

选择适合你的安装方式：全局安装或项目依赖

# 全局安装（推荐 CLI 使用）
npm install -g @pengzi/kms

# 项目依赖安装
npm install @pengzi/kms
2

连接数据库

配置 MongoDB 连接，生产环境建议使用 TLS

const { KMSClient } = require('@pengzi/kms');

const kms = new KMSClient({
  connectionString: 'mongodb://localhost:27017',
  databaseName: 'kms'
});

await kms.connect();
3

创建项目

创建一个新的项目来组织你的密钥

kms.setCurrentUser('admin-user');

const project = await kms.createProject(
  'my-application',
  'StrongPassword123!',
  {
    environment: 'production',
    team: 'engineering'
  }
);
4

创建密钥

创建你的第一个加密密钥

const { KeyType } = require('@pengzi/kms');

const key = await kms.createKey(
  project.projectId,
  'StrongPassword123!',
  {
    keyName: 'mongodb-primary',
    keyType: KeyType.MONGODB,
    value: 'mongodb://user:pass@localhost:27017/mydb',
    tags: ['production', 'database']
  }
);
5

获取密钥

需要时解密获取密钥值

const keyValue = await kms.getKey(
  project.projectId,
  'StrongPassword123!',
  key.keyId
);

console.log('连接字符串:', keyValue.value);

安全机制详解

多层安全防护体系，确保密钥全生命周期安全

🔐

第 1 层：密钥加密存储

AES-256-GCM 认证加密，PBKDF2-SHA256 密钥派生

  • 每个项目独立主密钥
  • 密钥值永不以明文存储
  • 加密包含完整性验证
👮

第 2 层：访问控制

基于角色的访问控制（RBAC）

  • Admin - 所有权限
  • Operator - 创建、读取、更新
  • Developer - 读取密钥
  • Readonly - 列出密钥
  • Auditor - 查看审计日志
📝

第 3 层：审计日志

完整记录所有操作，支持安全监控和合规审计

  • 所有密钥操作记录
  • 用户操作追踪
  • 权限拒绝事件
  • 时间戳和操作者信息
🌐

第 4 层：传输安全

支持 TLS/SSL 加密传输，防止中间人攻击

  • MongoDB Atlas 自动 TLS
  • 自建 MongoDB TLS 配置
  • 证书验证和主机名检查
🔒

第 5 层：密码安全

Bcrypt 哈希存储，API 密钥安全生成

  • Bcrypt 加密（salt rounds: 10）
  • API 密钥哈希存储
  • 主密码强度要求
方面 传统方式 KMS 方式
密钥存储 ✗ 环境变量、配置文件（明文） ✓ 数据库加密存储
访问控制 ✗ 无限制或简单密码 ✓ RBAC 权限管理
审计日志 ✗ 无记录或手动记录 ✓ 完整自动记录
密钥轮换 ✗ 手动更新配置文件 ✓ API 支持
传输安全 ✗ 可能明文传输 ✓ TLS 加密传输
合规性 ✗ 难以满足合规要求 ✓ 完整审计追踪

API 使用指南

详细的 API 调用示例和最佳实践

基础 API 操作

1

初始化客户端

const { KMSClient } = require('@pengzi/kms');

const kms = new KMSClient({
  connectionString: 'mongodb://localhost:27017',
  databaseName: 'kms'
});

await kms.connect();
kms.setCurrentUser('user-id');
2

创建项目

const project = await kms.createProject(
  'my-app',
  'master-password-123',
  { env: 'production' }
);
3

创建密钥

const { KeyType } = require('@pengzi/kms');

const key = await kms.createKey(
  project.projectId,
  'master-password-123',
  {
    keyName: 'redis-cache',
    keyType: KeyType.REDIS,
    value: 'redis://localhost:6379',
    tags: ['cache']
  }
);
4

获取密钥

const keyValue = await kms.getKey(
  project.projectId,
  'master-password-123',
  key.keyId
);

console.log(keyValue.value);
5

断开连接

await kms.disconnect();

高级 API 操作

👥

用户管理

创建用户和管理角色权限

// 创建用户
const user = await kms.createUser(
  projectId,
  {
    username: 'dev1',
    password: 'Pass123!',
    roles: ['developer']
  }
);

// 授予角色
await kms.grantRole(
  projectId,
  user.userId,
  'admin'
);
📊

密钥查询

分页查询和过滤密钥

// 分页查询
const { keys, total } = await kms.listKeys(
  projectId,
  { tags: ['production'] },
  { page: 1, limit: 20 }
);
🔍

审计日志

查询和分析操作记录

// 查询日志
const logs = await kms.getAuditLogs(
  projectId,
  {
    action: 'CREATE_KEY',
    limit: 100
  }
);
✏️

密钥更新

更新密钥属性和值

// 更新密钥
await kms.updateKey(
  projectId,
  masterPassword,
  keyId,
  {
    description: '新的描述',
    tags: ['updated']
  }
);

CLI 工具使用

💡 提示

CLI 工具提供交互式菜单，无需编写代码即可管理密钥。

1

启动 CLI

# 全局安装后
kms

# 或使用 npx
npx @pengzi/kms
2

创建项目

选择：项目管理 → 创建新项目，输入项目名称和主密码

3

创建密钥

选择：密钥管理 → 创建密钥，选择项目、输入密钥信息

4

获取密钥

选择：密钥管理 → 获取密钥值，输入主密码查看解密后的密钥值

5

查看日志

选择：审计日志 → 查看最近日志，查看所有操作记录

⚠️ 注意

CLI 工具显示密钥值时，请确保周围没有他人，密钥值会在终端明文显示。
