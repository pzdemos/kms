5 分钟快速开始

1. 安装 KMS

选择适合你的安装方式：

// 全局安装（推荐 CLI 使用）
npm install -g @pengzi/kms

// 项目依赖安装
npm install @pengzi/kms

2. 连接数据库

配置 MongoDB 连接，生产环境建议使用 TLS：

const { KMSClient } = require('@pengzi/kms');

const kms = new KMSClient({
  connectionString: 'mongodb://localhost:27017',
  databaseName: 'kms'
  // 生产环境添加 TLS 配置
  // connectionOptions: { tls: true }
});

await kms.connect();

3. 创建项目

创建一个新的项目来组织你的密钥：

kms.setCurrentUser('admin-user');

const project = await kms.createProject(
  'my-application',      // 项目名称
  'StrongPassword123!',  // 主密码（请妥善保管）
  {
    environment: 'production',
    team: 'engineering'
  }
);

4. 创建密钥

创建你的第一个加密密钥：

const { KeyType } = require('@pengzi/kms');

const key = await kms.createKey(
  project.projectId,
  'StrongPassword123!',
  {
    keyName: 'mongodb-primary',
    keyType: KeyType.MONGODB,
    value: 'mongodb://user:pass@localhost:27017/mydb',
    tags: ['production', 'database'],
    description: '主数据库连接'
  }
);

5. 获取密钥

需要时解密获取密钥值：

const keyValue = await kms.getKey(
  project.projectId,
  'StrongPassword123!',
  key.keyId
);

console.log('连接字符串:', keyValue.value);
// 输出: mongodb://user:pass@localhost:27017/mydb

6. 查看审计日志

追踪所有密钥操作：

const logs = await kms.getAuditLogs(project.projectId, {
  limit: 20
});

logs.logs.forEach(log => {
  console.log(`${log.timestamp} - ${log.action}`);
});
🎉 恭喜！你已经完成了 KMS 的基本使用流程。

安全机制详解

🔐 第 1 层：密钥加密存储

加密算法：AES-256-GCM（认证加密）

密钥派生：PBKDF2-SHA256（100,000 次迭代）

安全特性：

  • 每个项目独立的主密钥
  • 密钥值永远不以明文存储
  • 加密包含完整性验证
  • 密钥泄露只影响单个项目

👮 第 2 层：访问控制

权限模型：基于角色的访问控制（RBAC）

角色类型：

  • Admin：所有权限
  • Operator：创建、读取、更新密钥
  • Developer：读取密钥
  • Readonly：列出密钥
  • Auditor：查看审计日志

权限验证：每次操作都进行权限检查

📝 第 3 层：审计日志

记录内容：

  • 所有密钥操作（创建、读取、更新、删除）
  • 用户操作（登录、创建、角色变更）
  • 权限拒绝事件
  • 时间戳和操作者信息

用途：安全监控、合规审计、故障排查

🌐 第 4 层：传输安全

TLS/SSL 支持：

  • MongoDB Atlas 自动 TLS
  • 自建 MongoDB 支持 TLS 配置
  • 证书验证和主机名检查
  • 防止中间人攻击

推荐：生产环境必须使用 TLS 连接

🔒 第 5 层：密码安全

密码哈希：Bcrypt（salt rounds: 10）

API 密钥：哈希存储，不可逆

主密码要求：

  • 至少 8 个字符
  • 包含大小写字母、数字
  • 建议定期轮换（180 天）

安全对比：传统方式 vs KMS

方面 传统方式 KMS 方式
密钥存储 ✗ 环境变量、配置文件（明文） ✓ 数据库加密存储
访问控制 ✗ 无限制或简单密码 ✓ RBAC 权限管理
审计日志 ✗ 无记录或手动记录 ✓ 完整自动记录
密钥轮换 ✗ 手动更新配置文件 ✓ API 支持
传输安全 ✗ 可能明文传输 ✓ TLS 加密传输
合规性 ✗ 难以满足合规要求 ✓ 完整审计追踪

API 使用指南

基础 API 操作

1

初始化客户端

const { KMSClient } = require('@pengzi/kms');

const kms = new KMSClient({
  connectionString: 'mongodb://localhost:27017',
  databaseName: 'kms'
});

await kms.connect();
kms.setCurrentUser('user-id');
2

创建项目

const project = await kms.createProject(
  'my-app',              // 项目名称
  'master-password-123', // 主密码
  { env: 'production' }   // 元数据
);
3

创建密钥

const { KeyType } = require('@pengzi/kms');

const key = await kms.createKey(
  project.projectId,
  'master-password-123',
  {
    keyName: 'redis-cache',
    keyType: KeyType.REDIS,
    value: 'redis://localhost:6379',
    tags: ['cache']
  }
);
4

获取密钥

const keyValue = await kms.getKey(
  project.projectId,
  'master-password-123',
  key.keyId
);

console.log(keyValue.value);
5

断开连接

await kms.disconnect();
